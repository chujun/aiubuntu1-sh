# VMware Workstation Pro 迁移至 Hyper-V 方案

> 迁移日期: 2026-05-01
> 原VMware版本: VMware Workstation Pro 25H2 (build 24995812)
> 目标平台: Windows 11 Pro Hyper-V

---

## 背景

VMware Workstation Pro 25H2 虚拟机启动持续失败（错误1063: VMAuthdService无法正常启动），问题根源为安全软件（奇安信、腾讯电脑管家）修改了系统配置导致vmware-authd.exe无法正常启动，即使完全卸载重装也无法恢复。

决定迁移至Hyper-V作为替代方案。

## 现有虚拟机清单

| VM名称 | 类型 | CPU | 内存 | 磁盘大小 | 用途 |
|--------|------|------|------|----------|------|
| aiubuntus1 | Ubuntu服务器 | 2核 | 4GB | 30GB | 通用Ubuntu服务器 |
| aiubuntudesktop1 | Ubuntu桌面 | 2核 | 4GB | 40GB | 图形界面Ubuntu |
| aiubuntudesktop2 | Ubuntu桌面 | 2核 | 4GB | 40GB | 图形界面Ubuntu |
| k8s-control1 | Ubuntu服务器 | 4核 | 8GB | 60GB | Kubernetes控制节点 |

**VMware VMs 位置**: `D:\repository\vmware\`

**磁盘格式**: VMware vmdk ( monolithic flat格式 )

---

## 方案结论

**不需要VMware Workstation Pro软件**。Hyper-V是Windows 11 Pro内置功能，直接启用即可。

---

## Phase 1: 启用 Hyper-V

### 1.1 通过 DISM 启用（推荐）

以**管理员身份**运行PowerShell，执行：

```powershell
dism /online /enable-feature /featurename:Microsoft-Hyper-V-All /all /norestart
```

### 1.2 通过图形界面启用

1. 打开**控制面板** → **程序** → **启用或关闭Windows功能**
2. 勾选**Hyper-V**（包括 Hyper-V 管理工具和 Hyper-V 平台）
3. 点击确定，系统会自动安装

### 1.3 重启系统

启用Hyper-V后**必须重启系统**使配置生效。

### 1.4 验证安装

重启后以管理员身份运行PowerShell：

```powershell
# 检查Hyper-V服务状态
Get-Service vm*

# 验证Hyper-V管理工具
Get-Command *vm*
```

预期输出应包含 `vmcompute`, `vmms` 等服务。

## Phase 2: 准备磁盘转换

### 2.1 转换工具选择

需要将 `.vmdk` 格式转换为 Hyper-V 原生的 `.vhdx` 格式。

| 工具 | 优点 | 缺点 |
|------|------|------|
| **StarWind V2V Converter** | 免费、简单可靠 | 需要下载安装 |
| Microsoft MVMC | 官方工具 | 已停止维护 |
| qemu-img | 开源 | 命令行操作 |

**推荐使用 StarWind V2V Converter**

### 2.2 下载 StarWind V2V Converter

下载地址: https://www.starwindsoftware.com/starwind-v2v-converter

选择免费版本（StarWind V2V Converter Free）下载。

### 2.3 快照处理（针对k8s-control1）

k8s-control1 存在快照文件，需要先删除快照：

1. 打开VMware Workstation（如果能启动）
2. 对k8s-control1右键 → 快照 → 删除所有快照
3. 等待快照合并完成

**注意**: 如果VMware无法启动，快照将保留在原状态，转换时可能需要额外处理。

## Phase 3: 转换 VMware 磁盘

### 3.1 StarWind V2V Converter 使用步骤

1. 启动StarWind V2V Converter
2. 选择 **Local file**（本地文件）模式
3. 点击 **Browse** 浏览到 `D:\repository\vmware\[vm-name]\*.vmdk`
4. Destination Image Format 选择 **VHDX**
5. 点击 **Next** → 选择输出路径（建议保存到 `D:\repository\hyperv\`）
6. 点击 **Convert** 开始转换

### 3.2 各VM转换清单

| VM名称 | 源vmdk路径 | 输出vhdx路径 |
|--------|-----------|--------------|
| aiubuntus1 | D:\repository\vmware\aiubuntus1\aiubuntus1-0.vmdk | D:\repository\hyperv\aiubuntus1.vhdx |
| aiubuntudesktop1 | D:\repository\vmware\aiubuntudesktop1\*.vmdk | D:\repository\hyperv\aiubuntudesktop1.vhdx |
| aiubuntudesktop2 | D:\repository\vmware\aiubuntudesktop2\*.vmdk | D:\repository\hyperv\aiubuntudesktop2.vhdx |
| k8s-control1 | D:\repository\vmware\k8s-control1\*.vmdk | D:\repository\hyperv\k8s-control1.vhdx |

### 3.3 转换注意事项

- aiubuntus1 是 monolithic flat 格式（单文件），转换相对简单
- k8s-control1 有多个快照文件（Snapshot1-3），需先删除快照
- 转换过程可能需要10-30分钟，取决于磁盘大小
- 确保目标磁盘有足够空间（vhdx通常是精简分配，实际占用略大于原vmdk）

## Phase 4: 创建 Hyper-V 虚拟机

### 4.1 创建目录结构

```powershell
# 创建Hyper-V VMs存放目录
New-Item -ItemType Directory -Path "D:\repository\hyperv" -Force
```

### 4.2 创建虚拟交换机

在创建VM前，需要先创建虚拟交换机：

1. 打开 **Hyper-V管理器**
2. 右键点击主机 → **虚拟交换机管理器**
3. 选择 **外部** → 点击 **创建虚拟交换机**
4. 名称填写 `External Switch`
5. 外部网络选择您的物理网卡
6. 勾选 **允许管理操作系统共享此网络适配器**
7. 点击确定

### 4.3 创建 aiubuntus1 VM

```powershell
# 创建VM（不附加磁盘）
New-VM -Name "aiubuntus1" -MemoryStartupBytes 4GB -Generation 2 -Path "D:\repository\hyperv" -SwitchName "External Switch"

# 设置CPU核心数
Set-VMProcessor -VMName "aiubuntus1" -Count 2

# 禁用安全启动（Ubuntu通常需要）
Set-VM -VMName "aiubuntus1" -SecureBootTemplate "MicrosoftUEFI"

# 添加转换后的磁盘
Add-VMHardDiskDrive -VMName "aiubuntus1" -Path "D:\repository\hyperv\aiubuntus1.vhdx"
```

### 4.4 创建其他VM

```powershell
# aiubuntudesktop1
New-VM -Name "aiubuntudesktop1" -MemoryStartupBytes 4GB -Generation 2 -Path "D:\repository\hyperv" -SwitchName "External Switch"
Set-VMProcessor -VMName "aiubuntudesktop1" -Count 2
Add-VMHardDiskDrive -VMName "aiubuntudesktop1" -Path "D:\repository\hyperv\aiubuntudesktop1.vhdx"

# aiubuntudesktop2
New-VM -Name "aiubuntudesktop2" -MemoryStartupBytes 4GB -Generation 2 -Path "D:\repository\hyperv" -SwitchName "External Switch"
Set-VMProcessor -VMName "aiubuntudesktop2" -Count 2
Add-VMHardDiskDrive -VMName "aiubuntudesktop2" -Path "D:\repository\hyperv\aiubuntudesktop2.vhdx"

# k8s-control1（配置更高）
New-VM -Name "k8s-control1" -MemoryStartupBytes 8GB -Generation 2 -Path "D:\repository\hyperv" -SwitchName "External Switch"
Set-VMProcessor -VMName "k8s-control1" -Count 4
Add-VMHardDiskDrive -VMName "k8s-control1" -Path "D:\repository\hyperv\k8s-control1.vhdx"
```

## Phase 5: 启动与调试

### 5.1 首次启动检查

每个VM首次启动时检查：

1. **网络连接**: VM内部执行 `ping 8.8.8.8` 确认外网连通
2. **SSH访问**: 确认SSH服务正常运行
3. **数据完整性**: 检查关键文件和配置是否完整

### 5.2 常见问题与解决

#### 问题1: Secure Boot Violation

**错误**: VM启动时显示 "Secure Boot Violation"

**解决**:
1. 在Hyper-V管理器中选中VM
2. 点击 **设置** → **安全** → 取消勾选 **启用安全启动**
3. 点击确定，重新启动VM

#### 问题2: 启动设备找不到

**错误**: "No boot device found"

**解决**:
1. 设置 → **固件** → 调整引导顺序
2. 确保硬盘在引导顺序中排在第一位
3. 如果使用UEFI，确保开启了CSM兼容模块

#### 问题3: 网络不通

**解决**:
1. 检查虚拟交换机是否正确创建
2. 在Hyper-V管理器中检查VM网络适配器设置
3. 尝试使用 **默认开关（Default Switch）** 而非外部交换机

#### 问题4: 分辨率问题（桌面版）

**解决**:
1. 安装Hyper-V集成服务（通常已内置在Linux内核）
2. 在Ubuntu中安装: `sudo apt install linux-tools-virtual`

## 替代方案：VirtualBox

如果Hyper-V迁移遇到困难，可考虑VirtualBox作为过渡方案。

### VirtualBox 优点

- **可直接使用vmdk文件**，无需格式转换
- 配置简单，界面友好
- 与VMware Workstation操作习惯接近

### VirtualBox 缺点

- 需要安装额外软件（非Windows内置）

### VirtualBox 下载

https://www.virtualbox.org/wiki/Downloads

## 迁移后清理

### 卸载VMware Workstation Pro（可选）

确认Hyper-V正常工作后，可选择卸载VMware：

1. 打开 **控制面板** → **程序和功能**
2. 找到 **VMware Workstation Pro**
3. 点击 **卸载**
4. 可选择保留或删除虚拟机文件

### 保留VMware文件

VMware虚拟机文件（.vmdk, .vmx）可保留在原位置，以便日后需要时进行其他转换。
