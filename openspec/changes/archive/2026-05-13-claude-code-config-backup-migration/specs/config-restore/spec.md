## 新增需求

### 需求：一键恢复备份
系统应提供恢复脚本，允许用户通过单个命令从 GitHub 仓库恢复所有 Claude Code 配置。

#### 场景：在新系统上完整恢复
- **当** 用户在新系统上运行 `restore-backup.sh`
- **那么** 脚本应将备份仓库克隆到 `~/.claude-config-backup/`
- **并且** 脚本应将所有配置文件复制到 `~/.claude/` 中的正确位置
- **并且** 恢复应在不覆盖现有未跟踪文件的情况下完成

### 需求：选择性恢复能力
系统应允许用户恢复特定目录，而不是完整恢复。

#### 场景：恢复特定目录
- **当** 用户运行 `restore-backup.sh --dir skills`
- **那么** 只有 `skills/` 目录会被恢复
- **并且** 其他目录保持不变

### 需求：恢复前置检查
系统应在尝试恢复前验证前置条件。

#### 场景：GitHub CLI 未安装
- **当** `gh` CLI 未安装时
- **那么** 恢复脚本应显示错误消息说明需要 `gh`
- **并且** 恢复应以非零退出码中止

#### 场景：GitHub 未认证
- **当** GitHub CLI 未认证（`gh auth status` 失败）
- **那么** 脚本应提示用户运行 `gh auth login`
- **并且** 在认证成功前恢复应中止

### 需求：恢复保留 git 仓库
系统应在恢复后保留 git 仓库，以便将来进行增量同步。

#### 场景：恢复后 git 仓库保留
- **当** 恢复成功完成后
- **那么** `~/.claude-config-backup/.git/` 目录应存在
- **并且** 后续 Claude Code 会话应能够推送更新到远程

### 需求：覆盖时提示确认
系统应在覆盖现有配置文件前请求确认。

#### 场景：覆盖时的确认提示
- **当** 恢复检测到 `~/.claude/` 中存在现有文件
- **那么** 脚本应提示用户确认
- **并且** 如果用户拒绝，恢复应中止
- **并且** 如果用户确认，现有文件应被覆盖
