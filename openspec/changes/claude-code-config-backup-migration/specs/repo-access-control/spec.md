## 新增需求

### 需求：仓库可见性验证
系统应在同步前验证仓库可见性，拒绝同步到公开仓库。

#### 场景：公开仓库的同步被阻止
- **当** 目标仓库是公开的（`private: false`）
- **那么** 同步脚本应输出错误消息："ERROR: Refusing to sync to PUBLIC repository"
- **并且** 同步应以退出码 1 中止

#### 场景：允许同步到私有仓库
- **当** 目标仓库是私有的（`private: true`）
- **那么** 同步脚本应继续执行同步
- **并且** 不应引发任何错误

### 需求：使用 GitHub API 进行验证
系统应使用 GitHub CLI (`gh`) 通过 GitHub API 查询仓库可见性。

#### 场景：查询仓库隐私状态
- **当** 验证被触发时
- **那么** 脚本应执行 `gh api repos/{owner}/{repo} --jq '.private'`
- **并且** 解析响应以确定可见性
- **并且** 优雅地处理 API 错误

### 需求：验证失败不阻塞退出
系统应优雅地处理验证失败，不阻塞 Claude Code 退出。

#### 场景：验证期间 API 错误
- **当** 可见性检查期间 GitHub API 返回错误时
- **那么** 错误应记录到 `~/.claude/my-backups/my-sync.log`
- **并且** 系统应为安全起见默认拒绝同步
- **并且** Claude Code 退出不应被阻塞
