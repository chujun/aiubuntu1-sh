## 新增需求

### 需求：退出时自动备份
系统应在 Claude Code 会话结束时（SessionEnd）自动将配置同步到 GitHub 仓库。

#### 场景：会话结束时成功备份
- **当** Claude Code 会话结束时
- **那么** SessionEnd 钩子应触发备份脚本
- **并且** 所有配置的目录应同步到远程仓库
- **并且** 同步应使用 git 增量管理

#### 场景：网络故障时处理备份
- **当** 同步尝试时网络不可用
- **那么** 系统应将失败记录到 `~/.claude/backups/sync.log`
- **并且** 系统应正常结束会话，不阻塞退出
- **并且** 下次成功同步时应包含所有待处理的更改

### 需求：备份内容范围
系统应同步 `~/.claude/` 目录下的所有内容，包括：
- `settings.json` - 全局配置
- `AGENTS.md` - 自定义 agent 定义
- `rules/` - 编码规范和规则
- `skills/` - 技能库
- `memory/` - 全局记忆文件
- `hooks/` - 钩子配置
- `mcp-configs/` - MCP 服务配置
- `projects/` - 项目配置和完整会话历史

#### 场景：所有路径都被同步
- **当** 备份被触发时
- **那么** 每个路径应同步到远程仓库
- **并且** 同步应保持目录结构

### 需求：增量同步
系统应使用 git 执行增量同步，只同步有变化的文件。

#### 场景：只同步有变化的文件
- **当** 备份触发后自上次同步以来没有文件变化
- **那么** git commit 应报告 "nothing to commit"
- **并且** 不应执行推送

#### 场景：检测并同步新文件和修改
- **当** 在 `projects/` 中创建或修改了文件
- **那么** git 应检测这些变化
- **并且** git add -A 应暂存所有变化
- **并且** git commit 应将它们包含在备份中

### 需求：冲突检测与处理
系统应在推送前检测冲突，无法自动解决时提示用户手动处理。

#### 场景：自动合并成功
- **当** 执行 `git pull --rebase` 时没有冲突
- **那么** 系统应继续执行 `git push`
- **并且** 同步应成功完成

#### 场景：冲突无法自动解决
- **当** 执行 `git pull --rebase` 时产生无法自动解决的冲突
- **那么** 系统应将警告记录到 `~/.claude/backups/sync.log`
- **并且** 记录应包含 "unresolved conflicts - manual intervention required"
- **并且** 系统应中止同步，等待用户手动解决
- **并且** 会话应正常结束，不被阻塞

### 需求：备份日志
系统应维护备份操作日志，用于调试和问题追踪。

#### 场景：成功备份被记录
- **当** 备份成功完成时
- **那么** 条目 `[时间戳] Sync OK` 应追加到 `~/.claude/backups/sync.log`

#### 场景：失败备份被记录
- **当** 备份因网络或其他错误失败时
- **那么** 条目 `[时间戳] Sync FAILED, will retry next time` 应追加到日志

#### 场景：冲突警告被记录
- **当** 备份因冲突无法自动解决而中止时
- **那么** 条目 `[时间戳] Sync FAILED: unresolved conflicts - manual intervention required` 应追加到日志
